## Design Context

### Users

TypeScript developers — backend & full-stack — building real production systems where async error handling matters: payment processing, AI agent workflows, ETL, background jobs, HTTP handlers. They've felt the pain of `try/catch` losing type information and `error: unknown`. Many have tried or evaluated **Effect** and respect it, but find its method-chain syntax a leap from idiomatic async/await. They read docs in a dark IDE, often late, often while debugging.

### Brand Personality

**Pragmatic. Type-rigorous. Quietly confident.**

Voice is dry and direct, with a small dose of wit (the name is a pun; the tagline "*Good things come to those who a-wait-ly*" is intentionally cheeky). Never marketing-fluffy. Code speaks first; prose explains. Confident enough to compare itself to Effect by name without apologizing.

Emotional goal: relief — "oh, I can have the safety without learning a whole DSL."

### Aesthetic Direction

**Editorial-technical, dark-first.** Think the technical journal of a strongly opinionated software house, not a SaaS marketing page. Code is the visual hero. Annotations and inferred TypeScript types appear *as design elements*, not screenshots.

- **Theme**: dark-first (warm ink, not cool grey-blue), with a calmer light counterpart that mirrors the same intent
- **Typography**: Schibsted Grotesk for display + body (sharp, slightly editorial, not the Inter/Geist default), Geist Mono for code (already in repo). Big size & weight contrast, comfortable with italic for emphasis.
- **Color**: warm ink background (oklch ~0.16 with a subtle warm tint), bone/cream text, ONE saturated accent — warm amber for `ok` / emphasis, muted rust for `err`. Neutrals tinted toward the amber hue for cohesion. No cyan-on-dark, no purple→blue gradients, no gradient text.
- **Layout**: asymmetric, generous, magazine-grid. Numbered editorial sections (01 / 02 / 03). Leading numerals over icons. Full-bleed where it earns it.
- **Motion**: minimal and meaningful — one staggered hero reveal, subtle hover states, optional once-only "step unwrap" animation on the concept diagram. No scroll-jacking, no parallax.

**References (in spirit, not in pixel)**: Effect.website's confidence + density, Inngest's editorial restraint, Linear's typographic hierarchy.

**Anti-references**: any Starlight homepage that looks like Astro's, generic shadcn marketing pages, glass-morphism dashboard previews, gradient-text hero headlines, "feature card grid with rounded icon + heading + 2-line description" repeated four times.

### Design Principles

1. **Code is the art direction.** The diff between try/catch and awaitly should be the most striking thing on the page.
2. **TypeScript types are decoration.** Inferred error unions shown as annotations next to code beat any abstract illustration.
3. **One accent, used rarely.** Amber appears only at semantic moments (ok, primary CTA, hover). Rust only on err. Everything else is ink + cream.
4. **Editorial, not promotional.** Numbered sections, generous typography, prose written like a paragraph in a technical journal.
5. **Match the rigor of the library.** No filler. Every section earns its place. If a section isn't load-bearing, cut it.
